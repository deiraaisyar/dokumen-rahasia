# dokumen-rahasia

Submission version of a risk-register pipeline that processes Excel/PDF inputs and generates final outputs.

## Pipeline for Submission

Use the submission entry point:

- `run_pipeline_submission.py`

This runs 4 steps:

1. Extracting (`submission_pipeline/extracting.py`)
2. Preprocessing (`submission_pipeline/preprocessing.py`)
3. Generating outputs (`submission_pipeline/generating_outputs.py`)
4. Evaluating (`submission_pipeline/evaluating.py`)

## Requirements

- Python 3.10+
- Dependencies from `requirements.txt`
- `.env` with `GROQ_API_KEY`

Install dependencies:

```bash
python3 -m pip install -r requirements.txt
```

## Required Folder Layout (Repository Root)

Input sources:

- `inputs/` (main input folder used by extraction)
- `outputs/` (optional existing output files to re-extract)

Generated by pipeline:

- `extracted_inputs/`
- `extracted_outputs/`
- `preprocessed_inputs/`
- `preprocessed_outputs/`
- `generated_outputs/`

## How to Run

```bash
python3 run_pipeline_submission.py
```

## Final Generated Files

The final output files are written to `generated_outputs/`:

- `1. IVC DOE (Final).xlsx`
- `2. City of York Council (Final).xlsx`
- `3. Digital Security IT Sample Register (Final).xlsx`
- `4. Moorgate Crossrail Register (Final).xlsx`
- `5. Corporate Risk Register (Final).xlsx`
- `evaluation_doc1_doc3.csv`

## Notes

- If `outputs/` is empty, extraction from outputs is skipped.
- Preprocessing for `extracted_outputs/` runs once after input preprocessing is complete.
